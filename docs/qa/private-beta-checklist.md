# Vence 文思私测检查清单

目标：确认私测版在核心写作路径、AI 建议、安全失败场景和恢复能力上足够可靠。

## 测试前准备

- [ ] 使用 `scripts/package.ps1 -Configuration Release -Platform x64` 生成安装/发布产物。
- [ ] 在干净工作区目录中启动应用。
- [ ] 准备一个本地 Markdown 文件，包含标题、列表、代码块、Mermaid 和 LaTeX 片段。
- [ ] 准备一个无网络环境或可临时断网的测试环境。
- [ ] 准备一组无效 API key 和一组空 provider 配置。

## 文档基础流程

- [ ] 新建文档后，编辑区可输入中文、英文、标点和换行。
- [ ] 保存新文档后，Markdown 文件写入本地工作区。
- [ ] 关闭并重新打开应用后，最近文档列表仍能找到该文档。
- [ ] 打开本地 Markdown 文件后，正文内容、标题和大纲可正确展示。
- [ ] 编辑已有文档并保存后，磁盘文件内容更新。

## 历史记录与恢复

- [ ] 连续显式保存两次后，历史快照中出现两个版本。
- [ ] 恢复到上一版本后，当前正文回到旧内容。
- [ ] 恢复前后的 Markdown 文件路径不变。
- [ ] 恢复失败时，当前正文不被破坏。
- [ ] 自动保存草稿不会污染正式历史版本。

## AI 建议

- [ ] 手动触发语法检查后，右侧 AI 面板展示结构化建议。
- [ ] 润色建议只作为候选展示，不自动写入正文。
- [ ] 接受建议后，仅对应范围被替换。
- [ ] 拒绝建议后，正文保持不变，建议状态更新。
- [ ] 用户继续输入时，旧 AI job 被取消，不再覆盖新上下文。

## 失败与隐私场景

- [ ] 断网时触发 AI，界面显示可理解错误，不丢失正文。
- [ ] API key 配置错误时，界面显示 provider 配置错误，不重复无止境请求。
- [ ] AI 响应无效 JSON 时，建议被拒绝进入正文。
- [ ] 日志中不出现正文全文、API key 或 provider secret。
- [ ] 远程 AI 请求前，用户能看见 provider、模型和发送范围。

## Markdown 扩展渲染

- [ ] Mermaid 图表语法正确时可预览。
- [ ] Mermaid 语法错误时显示渲染失败提示，不影响编辑。
- [ ] LaTeX 公式语法正确时可预览。
- [ ] LaTeX 语法错误时显示渲染失败提示，不影响编辑。
- [ ] 图片路径无效时显示占位或错误状态，不阻塞保存。

## 性能与稳定性

- [ ] 打开 1 MB Markdown 文档时，编辑区在可接受时间内可交互。
- [ ] 连续输入 60 秒无明显卡顿。
- [ ] 多次保存不会生成损坏文件。
- [ ] 快速触发多次 AI 操作时，并发受限且 UI 仍可响应。
- [ ] 应用异常退出后，重新打开不会破坏 Markdown 文件。

## 私测反馈记录

- [ ] 记录测试人、系统版本、应用版本和工作区路径。
- [ ] 记录所有阻塞问题的复现步骤。
- [ ] 标注问题等级：阻塞、严重、一般、体验建议。
- [ ] 收集“是否更愿意继续写作”的主观反馈。
